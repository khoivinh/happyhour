## Follow-up batch (2026-04-22 1404)

- **Duplicate-city edit: tile A stuck yellow** (`client/src/components/digital-clock.tsx`). Root cause: in `CitySelector`'s `onSelect`, calling `setOpen(false)` only updated Radix's controlled `open` prop — it did NOT trigger the external `onOpenChange` callback (Radix suppresses that for controlled-prop updates from the consumer side). Result: parent's `isDropdownOpen` stayed `true`, keeping the yellow focus background on the editing tile whenever no zone swap happened (i.e. the duplicate-city path). Fix: call `onOpenChange(false)` explicitly alongside `setOpen(false)` in `onSelect`.

## Follow-up batch (2026-04-22 very late evening)

- **Search enhancements** (`client/src/lib/city-lookup.ts`). Two search issues were surfaced by the user and fixed together. (1) Relevance ranking: "San Francisco" used to return the CA entry last because results were returned in east-to-west GMT order — the Philippines, Argentina, and El Salvador entries (smaller by population, different timezones) appeared ahead of the CA one. Now `searchCities` scores each match (4 exact / 3 prefix / 2 substring / 1 other-field) and tiebreaks by population rank ascending, so the most-populous match wins for same-named cities. (2) Diacritic-insensitive matching: typing "sao" previously returned zero results because `.includes("sao")` on "são paulo" doesn't match across the accent. Query is now folded through NFD + combining-mark strip; the raw dataset's `city_ascii` field is pre-lowercased at build time into a new `TimezoneOption.nameAscii` field used for matching. Also added `rank` to `TimezoneOption` (original pop-sorted index, 0 = most populous) as the tiebreaker key. Commit `fdbd09b`.
- **Clock tile active-state layout shift** (`client/src/components/digital-clock.tsx:402`). User flagged that engaging a tile (drag or dropdown-open states) caused a subtle growth + content nudge, which also shifted the row height of the whole grid. Root cause: the base class had no border, and active states added `border border-[color]` — in a content-sized box that grows outer dimensions by 2px and shifts contents inward by 1px (box-sizing: border-box). Fix: put `border border-transparent` on the base so every state reserves 1px of border space; active states override color only. Added `border-color` to the transition property list so the color fades alongside the background. Commit `dc91187`. Learning also captured to the new cross-project `Design Craft` catalog (separate project, separate devlog).
